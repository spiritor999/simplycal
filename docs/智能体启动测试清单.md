# 智能体启动测试清单

## ✅ 测试前准备

### 1. 确认环境
```bash
# 检查 Python 版本
python3 --version  # 应该是 3.12+

# 检查工作目录
pwd  # 应该在 /workspace/projects
```

### 2. 检查文件
```bash
# 检查配置文件
ls -lh config/agent_llm_config.json

# 检查智能体代码
ls -lh src/agents/agent.py

# 检查游戏文件
ls -lh assets/math_game.html

# 检查启动脚本
ls -lh scripts/start_*.sh
ls -lh scripts/stop_all.sh
```

### 3. 检查依赖
```bash
# 检查 Python 依赖
pip list | grep -E "fastapi|uvicorn|langchain"

# 如果缺少依赖，安装
pip install -r requirements.txt
```

### 测试6: 智能体与游戏联动（完整流程）

#### 步骤
```bash
# 1. 启动服务
bash scripts/start_all.sh

# 2. 请求游戏链接
GAME_LINK=$(curl -s -X POST http://localhost:8000/run \
  -H "Content-Type: application/json" \
  -d '{"query": "开始游戏"}' | grep -oP '(?<=game_link": ")[^"]*')

echo "游戏链接: $GAME_LINK"

# 3. 检查游戏链接是否可访问
curl -I "$GAME_LINK"
# 应该返回 200 OK

# 4. 模拟游戏完成
curl -X POST http://localhost:8000/run \
  -H "Content-Type: application/json" \
  -d '{"query": "游戏已完成，得了85分，做对9题，难度中等"}'

# 5. 停止服务
bash scripts/stop_all.sh
```

#### 预期结果
- ✅ 智能体能返回游戏链接
- ✅ 游戏链接可以访问
- ✅ 智能体能正确处理游戏完成结果
- ✅ 智能体返回评价和建议

## 🔍 故障排查

### 问题1: 服务启动失败
```bash
# 查看详细日志
tail -f /tmp/agent.log
tail -f /tmp/game.log

# 检查配置
cat config/agent_llm_config.json
```

### 问题2: 端口被占用
```bash
# 查看占用进程
lsof -i :8000
lsof -i :8080

# 杀死进程
kill -9 <PID>

# 或使用其他端口
PORT=8001 bash scripts/start_agent_server.sh
```

### 问题3: 依赖缺失
```bash
# 重新安装依赖
pip install -r requirements.txt
```

### 问题4: 权限问题
```bash
# 添加执行权限
chmod +x scripts/*.sh
```
