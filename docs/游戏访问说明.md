# 游戏访问说明

## 问题说明

当前 Coze 对象存储的访问链接报错 "missing token"，这是因为需要通过 Coze 平台的认证才能访问。

## 解决方案

### 方案一：使用本地文件（推荐）

**适用场景：**
- 本地测试和开发
- 个人使用

**访问方式：**
1. 直接打开项目根目录下的 `game.html` 文件
2. 或者使用本地服务器（推荐）

**启动本地服务器：**
```bash
cd /workspace/projects
python scripts/start_game_server.py
```

服务器将在 `http://localhost:8080/game.html` 启动

---

### 方案二：使用智能体访问

**适用场景：**
- 通过 Coze 平台使用
- 与智能体交互

**操作步骤：**
1. 启动简便运算智能体
2. 对智能体说："开始游戏" 或 "我想玩游戏"
3. 智体会返回游戏链接

**返回的游戏链接：**
- 主链接：本地文件路径
- 备用链接：对象存储链接（可能需要认证）

**注意：**
- 如果对象存储链接不可用，可以使用本地文件方式
- 或者启动本地服务器：`python scripts/start_game_server.py`

---

### 方案三：部署到其他平台（可选）

如果需要公网可访问的链接，可以部署到以下平台：

#### 1. GitHub Pages
**步骤：**
1. 在 GitHub 仓库设置中启用 Pages
2. 选择 main 分支作为源
3. 访问链接：`https://spiritor999.github.io/simplycal/game.html`

**当前状态：**

#### 2. Netlify

## 如何解决 "missing token" 错误

### 错误信息
```xml
<Error>
<Code>AccessDenied</Code>
<Message>missing token</Message>
<RequestId>...</RequestId>
</Error>
```

### 原因
- Coze 对象存储需要通过 Coze 平台的认证
- 直接访问缺少必要的 token

### 解决方法

**方法1：使用本地文件**
```bash
# 直接打开
open game.html  # macOS
start game.html # Windows
xdg-open game.html # Linux
```

**方法2：启动本地服务器**
```bash
cd /workspace/projects
python scripts/start_game_server.py
# 然后访问 http://localhost:8080/game.html
```

**方法3：通过智能体访问**
1. 启动简便运算智能体
2. 对智能体说："开始游戏"
3. 使用智能体返回的链接
