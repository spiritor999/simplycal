# 阿里服务器部署指南 - 简便运算智能体

## 📋 部署概述

本指南介绍如何在阿里服务器上部署简便运算智能体，**完全独立于现有的 materials 项目**，互不影响。

## 🚀 快速部署（10分钟）

### 前提条件
- ✅ 已有阿里服务器
- ✅ 已安装 Docker 和 Docker Compose
- ✅ 有 Coze 平台的 API 密钥

### 步骤1：连接服务器

```bash
ssh root@your-server-ip
```

### 步骤2：创建项目目录

```bash
# 进入你的项目目录（如果已有）
cd /opt

# 克隆代码
git clone https://gitee.com/spiritor/simplecal.git

# 进入项目目录
cd simplecal
```

### 步骤3：配置环境变量

```bash
# 复制环境变量模板
cp .env.simplecal.example .env.simplecal

# 编辑环境变量
vim .env.simplecal
```

**配置内容：**
```bash
# Coze 平台配置（必需）
COZE_WORKLOAD_IDENTITY_API_KEY=your_api_key_here
COZE_INTEGRATION_MODEL_BASE_URL=https://api.coze.com

# 其他配置保持默认
PORT=8000
LOG_LEVEL=INFO
COZE_WORKSPACE_PATH=/app
```

### 步骤4：添加执行权限

```bash
chmod +x deploy-simplecal.sh
```

### 步骤5：运行部署脚本

```bash
./deploy-simplecal.sh
```

### 步骤6：验证部署

```bash
# 检查服务状态
docker-compose -f docker-compose-simplecal.yml ps

# 检查健康状态
curl http://localhost:8000/health

# 查看日志
docker-compose -f docker-compose-simplecal.yml logs -f agent
```

## 🌐 配置 Nginx（可选）

### 方案A：使用现有 Nginx（推荐）

修改现有的 `nginx/nginx.conf`，添加简便运算智能体的配置：

```bash
cd /path/to/materials
vim nginx/nginx.conf
```

在配置文件中添加：

```nginx
server {
    listen 80;
    server_name simplecal.yourdomain.com;

    # 重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name simplecal.yourdomain.com;

    # 使用现有的 SSL 证书
    ssl_certificate /etc/nginx/ssl/your-certificate.crt;
    ssl_certificate_key /etc/nginx/ssl/your-certificate.key;

    # 代理到智能体服务
    location / {
        proxy_pass http://simplecal_agent:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

**重要：需要将 `simplecal_agent` 加入到 `app-network` 网络**

修改 `docker-compose-simplecal.yml`：

```yaml
networks:
  simplecal-network:
    driver: bridge
  # 加入现有网络
  app-network:
    external: true
```

添加到 agent 服务的网络配置：

```yaml
agent:
  # ... 其他配置
  networks:
    - simplecal-network
    - app-network  # 加入现有网络
```

### 方案B：使用独立端口（简单）

不使用 Nginx，直接通过端口访问：

```
http://your-server-ip:8000
```

### 方案C：申请新的 SSL 证书

如果需要独立的域名和证书：

```bash
# 安装 Certbot
apt install certbot python3-certbot-nginx -y

# 申请证书
certbot --nginx -d simplecal.yourdomain.com
```


```bash

```

```bash

docker-compose -f docker-compose-simplecal.yml logs -f agent

# 查看最近 100 行
docker-compose -f docker-compose-simplecal.yml logs --tail=100 agent
```

```bash
docker-compose -f docker-compose-simplecal.yml ps
```

```bash
git pull

# 重新构建
docker-compose -f docker-compose-simplecal.yml build

# 重启服务
docker-compose -f docker-compose-simplecal.yml up -d
```

## 🔍 故障排查

### 问题1：端口被占用

```bash
# 检查端口占用
lsof -i :8000

# 修改 docker-compose-simplecal.yml 中的端口
# 例如：- "8001:8000"
```

### 问题2：服务无法启动

```bash
# 查看详细日志
docker-compose -f docker-compose-simplecal.yml logs agent

# 检查环境变量
cat .env.simplecal

# 检查配置文件
cat config/agent_llm_config.json
```

### 问题3：健康检查失败

```bash
# 手动检查
curl http://localhost:8000/health

# 进入容器检查
docker exec -it simplecal_agent bash
curl http://localhost:8000/health
exit
```

### 问题4：Nginx 代理失败

```bash
# 检查 Nginx 配置
docker exec -it materials_nginx nginx -t

# 检查网络连接
docker network inspect app-network

# 确保 simplecal_agent 在 app-network 中
docker network connect app-network simplecal_agent
```

## 🔒 安全建议

### 1. 限制访问

```yaml
# 在 docker-compose-simplecal.yml 中添加网络限制
agent:
  # ... 其他配置
  networks:
    simplecal-network:
      aliases:
        - agent
```

### 2. 配置防火墙

```bash
# 只允许特定 IP 访问 8000 端口
ufw allow from your-ip to any port 8000
```

### 3. 使用 HTTPS

参考"配置 Nginx"部分

### 4. 定期更新

```bash
# 定期更新镜像
docker-compose -f docker-compose-simplecal.yml pull
docker-compose -f docker-compose-simplecal.yml up -d
```
