# 智能体启动与部署指南

## 概述

本指南提供多种方式来启动和部署简便运算智能体服务，从本地测试到云服务器生产部署。

---

## 方案对比

| 方案 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **本地启动** | 开发测试 | 简单快速，无需配置 | 仅本地访问，无法公网访问 |
| **ngrok 内网穿透** | 快速演示 | 无需服务器，快速公网访问 | 速度较慢，不稳定 |
| **云服务器部署** | 生产环境 | 稳定可靠，可长期使用 | 需要购买服务器 |
| **Docker 部署** | 生产环境 | 环境一致，易于维护 | 需要 Docker 基础 |

---

## 方案一：本地启动（开发测试）

### 前置要求
- Python 3.12+
- pip

### 启动步骤

1. **给启动脚本添加执行权限**
```bash
chmod +x scripts/start_agent_server.sh
```

2. **启动智能体服务**
```bash
bash scripts/start_agent_server.sh
```

3. **访问服务**
   - 本地访问: http://localhost:8000
   - API 文档: http://localhost:8000/docs

### 自定义配置

```bash
# 使用自定义端口
PORT=8001 bash scripts/start_agent_server.sh

# 使用多进程（生产模式）
WORKERS=4 bash scripts/start_agent_server.sh
```

### 测试服务

```bash
# 健康检查
curl http://localhost:8000/health

# 调用智能体
curl -X POST http://localhost:8000/run \
  -H "Content-Type: application/json" \
  -d '{"query": "你好"}'
```

## 方案二：ngrok 内网穿透（快速演示）

### 前置要求
- 已启动本地智能体服务（方案一）
- ngrok 账号（免费版即可）

### 安装 ngrok

**macOS:**
```bash
brew install ngrok
ngrok config add-authtoken YOUR_AUTH_TOKEN
```

**Linux:**
```bash
curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc >/dev/null
echo 'deb https://ngrok-agent.s3.amazonaws.com buster main' | sudo tee /etc/apt/sources.list.d/ngrok.list
sudo apt update && sudo apt install ngrok
ngrok config add-authtoken YOUR_AUTH_TOKEN
```

**Windows:**
下载并安装: https://ngrok.com/download

### 启动步骤

1. **先启动智能体服务**
```bash
bash scripts/start_agent_server.sh
```

2. **打开新终端，启动 ngrok**
```bash
chmod +x scripts/start_ngrok.sh
bash scripts/start_ngrok.sh
```

3. **获取公网地址**
ngrok 会显示一个公网地址，例如：
```
Forwarding  https://xxxx-xx-xx-xx-xx.ngrok-free.app -> http://localhost:8000
```

4. **使用公网地址访问**
   - 智能体: https://xxxx-xx-xx-xx-xx.ngrok-free.app
   - API 文档: https://xxxx-xx-xx-xx-xx.ngrok-free.app/docs

### 注意事项
- ⚠️ ngrok 免费版地址会变化，每次重启后需要更新
- ⚠️ 免费版有连接数和流量限制
- ⚠️ 适合快速测试，不适合长期生产使用

## 方案三：云服务器部署（生产环境）

### 推荐云服务器

| 提供商 | 配置建议 | 价格（月） |
|--------|----------|-----------|
| 阿里云 | 2核4G, 5M带宽 | ¥50-100 |
| 腾讯云 | 2核4G, 5M带宽 | ¥50-100 |
| 华为云 | 2核4G, 5M带宽 | ¥50-100 |

### 部署步骤（以 Ubuntu 为例）

#### 1. 连接到服务器

```bash
ssh root@your-server-ip
```

#### 2. 安装必要软件

```bash
# 更新系统
apt update && apt upgrade -y

# 安装 Python 3.12
apt install software-properties-common -y
add-apt-repository ppa:deadsnakes/ppa -y
apt update
apt install python3.12 python3.12-venv python3-pip git -y

# 安装 Nginx（反向代理）
apt install nginx -y

# 安装 Supervisor（进程管理）
apt install supervisor -y
```

#### 3. 克隆代码

```bash
cd /opt
git clone https://gitee.com/spiritor/simplecal.git
cd simplecal
```

#### 4. 创建虚拟环境

```bash
python3.12 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

#### 5. 配置环境变量

```bash
# 创建环境变量文件
cat > .env << EOF
COZE_WORKSPACE_PATH=/opt/simplecal
COZE_INTEGRATION_MODEL_BASE_URL=https://your-model-url
COZE_WORKLOAD_IDENTITY_API_KEY=your-api-key
EOF

# 加载环境变量
source .env
```

#### 6. 测试启动

```bash
# 测试智能体服务
python -m uvicorn src.main:app --host 0.0.0.0 --port 8000

# 按 Ctrl+C 停止
```

#### 7. 配置 Supervisor（守护进程）

```bash
# 创建配置文件
cat > /etc/supervisor/conf.d/simplecal-agent.conf << EOF
[program:simplecal-agent]
command=/opt/simplecal/venv/bin/uvicorn src.main:app --host 0.0.0.0 --port 8000
directory=/opt/simplecal
user=root
autostart=true
autorestart=true
stderr_logfile=/var/log/supervisor/simplecal-agent.err.log
stdout_logfile=/var/log/supervisor/simplecal-agent.out.log
environment=COZE_WORKSPACE_PATH="/opt/simplecal"
EOF

# 重新加载配置
supervisorctl reread
supervisorctl update

# 启动服务
supervisorctl start simplecal-agent

# 查看状态
supervisorctl status
```

#### 8. 配置 Nginx（反向代理 + HTTPS）

```bash
# 创建配置文件
cat > /etc/nginx/sites-available/simplecal << EOF
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto \$scheme;

        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade \$http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
EOF

# 启用配置
ln -s /etc/nginx/sites-available/simplecal /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重启 Nginx
systemctl restart nginx

# 配置防火墙
ufw allow 'Nginx Full'
```

#### 9. 配置 HTTPS（使用 Let's Encrypt）

```bash
# 安装 Certbot
apt install certbot python3-certbot-nginx -y

# 获取证书（会自动配置 HTTPS）
certbot --nginx -d your-domain.com

# 自动续期
certbot renew --dry-run
```

#### 10. 访问服务

```
http://your-domain.com
https://your-domain.com
https://your-domain.com/docs
```

## 方案四：Docker 部署

### 前置要求
- Docker
- Docker Compose

### 部署步骤

1. **构建并启动服务**

```bash
# 开发环境
docker-compose up

# 生产环境（后台运行）
docker-compose up -d
```

2. **查看日志**

```bash
docker-compose logs -f agent
docker-compose logs -f game-server
```

3. **停止服务**

```bash
docker-compose down
```

4. **访问服务**
   - 智能体: http://your-server-ip:8000
   - 游戏服务器: http://your-server-ip:8080

### 配置环境变量

创建 `.env` 文件：
```bash
COZE_INTEGRATION_MODEL_BASE_URL=https://your-model-url
COZE_WORKLOAD_IDENTITY_API_KEY=your-api-key
```

### 配置 Nginx（反向代理）

参考方案三的 Nginx 配置部分。

## 故障排查

### 问题1: 端口被占用

```bash
# 查看端口占用
lsof -i :8000

# 杀死进程
kill -9 <PID>
```

### 问题2: 服务无法启动

```bash
# 查看日志
tail -f /var/log/supervisor/simplecal-agent.err.log
tail -f /var/log/supervisor/simplecal-agent.out.log

# 或直接运行查看错误
python -m uvicorn src.main:app --host 0.0.0.0 --port 8000
```

### 问题3: 依赖缺失

```bash
# 重新安装依赖
pip install -r requirements.txt
```

### 问题4: 权限问题

```bash
# 添加执行权限
chmod +x scripts/*.sh
```
